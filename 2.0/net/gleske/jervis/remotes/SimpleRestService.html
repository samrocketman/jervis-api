        • static def apiFetch(URL api_url, Map http_headers = [:], String http_method = 'GET', String data = '')

          apiFetch() can be used to submit HTTP commands common to REST APIs. If http_headers does not contain a Content-Type header, then application/json is assumed.

          Extra behaviors are provided by special http_headers. These headers are not sent over HTTP but instead change the behavior of the response.

          Special HTTP Headers:
          Parse-JSON - For JSON-based APIs responses are automatically parsed. This setting can disable automatic parsing if set to false.
          Response-Code - Return the HTTP response code.
          Response-Headers - Return the HTTP response headers as an unmodifiable Map. The DELETE HTTP method requires setting Response-Code to false. The HEAD HTTP method implies returning response headers.
          Parameters:
          api_url - A URL of a REST endpoint in which to make an HTTP call.
          http_headers - HTTP headers to pass as part of the HTTP request. By default only Content-Type: application/json HTTP header will be set.
          http_method - The HTTP method or action to request from the server. Currently supported methods include: GET, POST, PUT, DELETE, and PATCH.
          Returns:
          If Content-Type HTTP header is application/json, then one of two types will be returned: HashMap or ArrayList (for JSON Object and JSON Array respectively). If any other Content-Type HTTP header used then a String will be returned and it will be left up to the caller to parse the result.
