Package: net.gleske.jervis.remotes

[Groovy] Class GitHub

    • net.gleske.jervis.remotes.GitHub
    • All Implemented Interfaces and Traits:
      JervisRemote, SimpleRestServiceSupport

      class GitHub
extends Object

      A simple class to interact with the GitHub v3 API for only the parts I need.

      Sample usage

      To run this example, clone Jervis and execute ./gradlew console to bring up a Groovy Console with the classpath set up.

      
import net.gleske.jervis.remotes.GitHub

def x = new GitHub()
println 'Print each branch.'
x.branches('samrocketman/jervis').each{ println it }
println 'Print the contents of .travis.yml from the main branch.'
println x.getFile('samrocketman/jervis','.travis.yml','main')

      • Property Detail

        • TokenCredential credential

          A credential for interacting with an external credential store. If this is defined or set, then gh_token is ignored and not used. Default: null

        • String gh_api

          URL to the GitHub v3 API. For GitHub Enterprise it should be gh_web + 'api/v3/'. Default: https://api.github.com/

        • String gh_clone

          The base clone URI in which repositories will be cloned. Default: https://github.com/

        • String gh_token

          The API token, which can be used to communicate with GitHub using authentication. Default: null

        • String gh_web

          URL to the GitHub web interface. Default: https://github.com/

        • Map headers

          Optional HTTP headers that can be added to every request.

        • List branches(String project)

          Get a list of branches for a project. This is meant to be a standard function for Jervis to interact with remotes. All remotes are required to have this function.

          Parameters:
          project - A GitHub project including the org. e.g. "samrocketman/jervis"
          Returns:
          A List where each element is a branch in the project.

        • def fetch(String path)

          Fetches a URL from GitHub API. This is mostly used by other functions to provide minimum functionality defined in JervisRemote. It can be used for general GitHub API communication.

          Parameters:
          path - A GitHub API path to fetch. The URL must return JSON content. e.g. user/repos.
          Returns:
          A Map or List from the parsed JSON response.

        • String getFile(String project, String file_path, String ref = '')

          Get the contents of a file from a project. This is meant to be a standard function for Jervis to interact with remotes. All remotes are required to have this function.

          returns:
          A String which contains the contents of the file requested.
          Parameters:
          project - A GitHub project including the org. e.g. "samrocketman/jervis"
          file_path - A path to a file relative to the root of the Git repository. e.g. ".travis.yml"
          ref - A git reference such as a branch, tag, or SHA1 hash. e.g. "main". This option is optional. If not specified the default branch is selected.

        • ArrayList getFolderListing(String project, String dir_path = '/', String ref = '')

          Parameters:
          project - A GitHub project including the org. e.g. samrocketman/jervis
          dir_path - A path to a directory relative to the root of the Git repository. This is optional. By default is / (the repository root).
          ref - A git reference such as a branch, tag, or SHA1 hash. e.g. main. This option is optional.
